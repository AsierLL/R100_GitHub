\doxysubsection{Detailed Description}
These methods provide a simple API to Crypto\+Auth chips. 

\hypertarget{md__c_1_2_users_2ivicente_2_desktop_2_b_r_a_n_c_h___a_c_c___w_i_f_i_2src_2_s_c_e_2calib_2_r_e_a_d_m_e}{}\doxysubsubsection{calib directory -\/ Purpose}\label{md__c_1_2_users_2ivicente_2_desktop_2_b_r_a_n_c_h___a_c_c___w_i_f_i_2src_2_s_c_e_2calib_2_r_e_a_d_m_e}
The purpose of this directory is to contain the files implementing the APIs for a basic interface to the core Crypto\+Auth\+Lib library.

High-\/level functions like these make it very convenient to use the library when standard configurations and defaults are in play. They are the easiest to use when developing examples or trying to understand the "{}flow"{} of an authentication operation without getting overwhelmed by the details.

This makes simple jobs easy and if you need more sophistication and power, you can employ the full power of the Crypto\+Auth\+Lib object model.

See the Doxygen documentation in cryptoauthlib/docs for details on the API of the calib commands.
